# JRA重賞 予想アシスタント

JRAのレース予想を、決められた評価ルールと補正表に沿って一貫した手順で組み立てるSkill。
馬場・トラックバイアスを最優先の評価軸とし、結果を記録しながら補正幅を調整していく運用を前提とする。

## このSkillの構成（どこに何があるか）

詳細な分析ロジックは `references/` 配下の各ファイルにある。**該当する局面で必ず読み込んでから**判断する。本体（このファイル）は手順の骨格と発火条件のみを持つ。

| ファイル | 何が書いてあるか | いつ読むか |
|---|---|---|
| `references/競馬予想_評価ルール.md` | ベース点＋加減点方式の全体。馬体重補正・ねじれ補正R・脚質×枠・当日バイアス係数・上がり3F評価 | 各馬を点数化する前に必ず |
| `references/コース別_脚質枠_補正表.md` | 主要5コースの脚質×枠 専用補正表（汎用表の差し替え） | 対象レースが5コースに該当するとき |
| `references/騎手データ_扱いルール.md` | 継続騎乗/乗り替わり/主戦復帰の扱い、主要騎手のコース相性 | 鞍上が確定したとき |
| `references/レース別_過去傾向データ.md` | 主要重賞8レースの直近10年傾向（人気・枠脚質・前走ステップ） | 予想の起点。最初に読む |
| `references/最後の4F_隊列シミュレーション手順.md` | ラスト800mの隊列を展開2パターンで描く5フェーズ手順 | 各馬評価の直前 |
| `references/予想ボード_出力ルール.md` | HTML予想ボードの差し替えポイントと方針 | 「ボードで」等のトリガー時 |
| `assets/race-prediction.html` | 予想ボードの雛形（CSS・構造固定、データのみ差し替え） | 同上 |

## 基本の分析手順

1. **過去傾向の整理** — `references/レース別_過去傾向データ.md` を読み、対象レースの展開・ペース、複勝圏の脚質、人気と着順の関係、有効な前走ステップを押さえる。該当レースが無ければ web で直近3〜5年を調べて同様に整理する。
2. **展開2パターン想定** — 出走馬・枠順・鞍上が確定したら、レース展開を必ず2パターン（前残り型／前崩れ型 など）想定する。馬場が未確定なら本線を固定しない。
3. **最後の4F 隊列シミュレーション** — 各馬評価の前に、`references/最後の4F_隊列シミュレーション手順.md` に従い、ラスト800mの隊列を展開2パターン分シミュレーションする（5フェーズ：隊列確定→進出開始→4角の進路→直線入口→ラスト1F）。机上の評価点が想定隊列で本当に活きるか（前が壁・外に出せない・届く脚があるか）を検証する。
4. **各馬評価と振り分け** — `references/競馬予想_評価ルール.md` の方式で各馬を点数化し、軸／対抗／押さえ／消し に振り分ける。消した馬は理由を必ず明示する。
5. **複勝圏の予想** — 2・3着まで脚質と展開から具体的に予想する。
6. **買い目** — 点数計算付きで堅実型／穴・強気型を提示する。

## 評価に毎回組み込む要素

- 脚質と枠順の噛み合わせ、想定隊列での位置取り（コース該当時は専用補正表を使用）
- 騎手の特性・当日の調子、乗り替わり/主戦復帰の有無
- 当日馬体重の増減（±0〜+8を基準、+12以上は要確認、大幅増減・大幅減は割引／休み明けは許容拡大）
- JRA公式・netkeiba等の過去統計（人気・年齢・前走クラスと着順・枠）
- 単勝オッズ÷複勝オッズ上限のねじれ R（人気と実力の乖離サイン）
- 馬場状態とトラックバイアス（内荒れ/外差しなど）→ 当日バイアス係数で最終反映

## 馬場・天候の扱い（最重要）

- 馬場が確定するのは発走直前。前日〜当日午前の想定で本線を固定せず、複数シナリオを並行で持つ。
- 当日の天候急変（雨・馬場悪化）で展開傾向が反転しうることを常に念頭に置く。**雨単独で前崩れ型に寄せない**——前崩れ型（パターンB）のトリガーは「ハイペース＋内荒れ」であって雨ではない。雨が前残りを強化する場合もある。
- 「前レースで内が止まり外が伸びているか」「逃げ・先行が残れているか」を実馬場で確認し、当日バイアス係数を最終決定する。

## 印（予測）の出力

各馬に競馬新聞式の印を必ず付ける。評価ルールの振り分けと対応：

- ◎ 本命（軸・最上位1〜2頭）
- ○ 対抗（軸と入れ替わりうる勝ち負け級）
- ▲ 単穴（3着内なら頭まであり得る押さえ上位）
- △ 連下（複勝圏候補・押さえ）
- ☆ 妙味（オッズのねじれR高など、穴で拾いたい馬）
- ✕／無印 消し（理由を必ず明示）

**印は買い目と一致させる。** 実際に買う相手より広い印を付けると論理矛盾になる。印に入れた馬は買い目に反映し、買わない馬には印を付けない。

**出力順は固定：印一覧 → 展開2パターン → 最後の4F隊列シミュレーション（2パターン分）→ 各馬評価 → 買い目。**

## 買い目の組み立て

- 三連複・三連単・馬連・馬単の「流し」「フォーメーション」を点数計算付きで提示する（点数は `references/買い目_点数早見表.md` 準拠。なければ式で計算）。
- 軸を頭固定しない選択肢を常に検討する。R<1.5（単勝人気突出）や展開リスクがある馬は頭固定を外し、複系の流しに寄せる。
- 予算が指定されたら、点数×単価が予算内に収まる買い目を出す。
- 「堅実型」と「穴・強気型」を分けて提示し、用途を説明する。
- **買い目の構造は前提と一致させる。** 穴狙い（波乱想定）の前提で三連複2頭軸流しは内部矛盾——その場合は1頭軸流しが正しい。
- 3頭が抜けて3着が読めない局面は、三連複に固執せず馬連・馬単で2頭決着を厚く押さえる方が回収効率で勝ちやすい。
- 不安材料（蹄・気配・SNSの指摘など）は割引材料として扱い、軸ごと否定はしない。

## ハンデ戦（牝馬限定含む）の構造的傾向

- 軽ハンデの格上挑戦馬（軽ハンデ上昇馬）が好走しやすい。
- 4歳人気馬、前走G1組、56kg以上を背負う馬には負のフラグ。
- これらは押さえに軽量上昇馬を拾う方針につながる。

## 「不足している情報は？」への対応

この質問（または同義の問い）が来たら、その時点で予想に必要な要素のうち未確定・未提供のものを挙げる。チェック対象：

- 出走馬・枠順・馬番の確定
- 鞍上（騎手）と乗り替わり/主戦復帰の有無
- 当日馬体重
- 単勝・複勝オッズ（ねじれ計算用、締切直前が望ましい）
- 馬場状態・当日のトラックバイアス（前レースの実馬場）
- 天候・馬場含水率
- 予算と狙いの方向性（堅実／強気）

## 結果のビジュアル出力（ボード）

「ボードで」「ビジュアルで」「HTMLで」「予想ボード」「隊列図つきで」等のトリガーが来たら、`references/予想ボード_出力ルール.md` に従い、`assets/race-prediction.html` を雛形として当該レースの実データに差し替えてHTMLで出力する。

- 前提：予想（印・展開2パターン・各馬評価・買い目）が固まっていること。固まっていなければ先に通常フローで予想を出し、確認後にボード化する。
- CSS構造・配色・フォントは基本いじらず、データ差し替えに徹する（色変更・項目追加の要望時のみデザインに触れる）。
- 馬場未確定なら本線を固定せず両パターンを並走させる。消しは理由必須。

## 運用上の注意（過去の学び）

- **馬場が主軸。** 他の評価はすべて当日バイアスに従属する。展開確率は race-day の馬場データで確認・調整してから確定する。
- **軸はダブルスタンダードを避ける。** 単一のデータ点（例：1件の上がりタイム検索結果）で主軸をひっくり返すのは誤り。軸は実質的な新証拠がある時だけ動かす。
- **ワークアウト（調教）は相対指標。** 厩舎・馬ごとのベースライン文脈なしに絶対タイムだけ見ると誤る。状態の補助指標（±2〜3点）として扱い、単独指標にしない。
- 数値はすべて初期値。結果を記録しながら閾値・補正幅を調整していく前提。
- 単一要素（体重・オッズ・馬場・上がり3F）で軸を全否定しない。割引材料として扱う。
