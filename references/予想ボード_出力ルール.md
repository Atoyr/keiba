# 予想ボード（HTML）出力ルール
 
`race-prediction.html` を雛形に、当該レースの実データへ差し替えてArtifactで出力するための早見メモ。
このファイルと `race-prediction.html` の2点をプロジェクトのナレッジに入れておく。
 
---
 
## トリガー（この語が来たら出力）
 
「ボードで」「ビジュアルで」「HTMLで」「予想ボード」「隊列図つきで」等。
→ その会話で組み立てた予想を、雛形の構造・CSSのままデータだけ差し替えてHTML Artifactで出す。
 
**前提：** 予想（印・展開2パターン・各馬評価・買い目）が会話内で固まっていること。
固まっていなければ先に通常フローで予想を出し、確認後にボード化する。
 
---
 
## 出力フロー（雛形のセクション順＝プロジェクト既定の順）
 
01 印一覧 → 02 展開2パターン（隊列図）→ 03 各馬評価 → 04 買い目
 
---
 
## 差し替えポイント一覧
 
### ヘッダー（.masthead）
- `.grade-tag` … 例 `G1` / `G2`（SAMPLE表記は消す）
- `.race-title` … レース名
- `.race-meta` … コース・距離・グレード・日付・想定馬場
- 雛形冒頭の `.swatch`（※ダミー注記）は本番では削除する
### 01 印一覧（.marks 内の .mark-card）
印ごとにクラスを付け替える：
| 印 | クラス | 色 |
|---|---|---|
| ◎本命 | `m-honmei` | 赤（枠太） |
| ○対抗 | `m-taikou` | 緑 |
| ▲単穴 | `m-tana` | ゴールド |
| △連下 | `m-renka` | スレート |
| ☆妙味 | `m-myoumi` | 紫 |
| ✕消し | `m-keshi` | 朱（淡色化） |
 
各カードの中身：`.num`=馬番 / `.glyph`=印 / `.horse`=馬名 / `.jockey`=騎手（継続/乗替）/ `.pace-bar`=脚質・枠 / `.stats` の `<b>`=評価点・R値。
頭数は増減可（カードを足す/減らす）。グリッドは自動折返し。
 
### 02 展開2パターン（隊列トラック図）
タブ2つ＝前残り型／前崩れ型（名称は当該レースの想定に合わせて変更）。
 
**隊列チップ `.chip` の座標が肝。トラックは上から見た俯瞰図：**
- `top%` … 前後位置。**0%=先頭、100%=最後方**
- `left%` … 内外位置。**8%=内ラチ際 〜 92%=大外**
- クラス：通常＝白 / `.hot`＝注目(伸びる)・ゴールド / `.keshi`＝消し・朱
- チップ表記は「印+馬番」または「馬番」（例 `◎⑤` / `⑪` / `✕②`）
各局面（START / 残り4F / ゴール前）で同じ馬を3回置き、top%・left%を変えることで「動き」を表現する。
- 前残り型：ゴール前で内前(low top / low left)に本線馬が残る
- 前崩れ型：ゴール前で外差し馬(high→low top / high left)が前に出る
`.phase-note`=各局面の一言 / `.note-line`=想定決着順。
 
### 03 各馬評価（details.horse）
1頭＝1ブロック。`open` を付けたものは最初から展開表示（◎は開いておくと良い）。
- `.sum-glyph`（色は印に合わせ inline style）/ `.sum-name`=馬番+馬名 / `.sum-sub`=脚質・枠・騎手 / `.sum-score .pt`=最終評価点
- `.calc` テーブル＝評価ルールの積み上げ（ベース点／馬体重補正／ねじれ補正R／上がり3F補正／脚質×枠×バイアス係数／最終評価点）
- 加点は `class="plus"`（緑）、減点は `class="minus"`（朱）
- `.reason`=採用理由。消しは `.reason.keshi`（赤帯）で理由必須
全頭載せなくてよい。◎○と消し代表は載せ、他は会話本文に委ねる運用でも可。
 
### 04 買い目（.bet-split）
左＝堅実型 / 右＝穴・強気型。
- `.bet-type`=券種（三連複 軸2頭流し 等）/ `.bet-formula`=買い目 / `.bet-cost`=点数×単価と小計
- `.bet-total`=合計（予算内に収める）/ `.warn`=トリガミ・馬場依存などの注意
- 予算指定があれば点数早見表で計算し合計を予算内に。`.sec-sub` に予算を記載
---
 
## 守ること（プロジェクト方針の踏襲）
- 馬場未確定なら本線を固定せず両パターンを並走させる（雛形は2タブ前提）
- 消しは理由を必ず明示
- 頭固定回避（R<1.5や展開リスク）の馬は買い目で複系流しに寄せる
- フッターの「馬場は発走直前確定／馬券は自己責任」注記は残す
## やらないこと
- CSS構造・配色・フォントは基本いじらない（データ差し替えに徹する）
- 色変更や項目追加の要望が来たときのみデザインに手を入れる
